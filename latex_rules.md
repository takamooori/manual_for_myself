# LaTeX原稿ルール（AI向けプロンプト）

以下は修士論文中間発表原稿（pLaTeX2e）の執筆ルールです。
TeXファイルの新規作成・編集・レビューを依頼する際は、このルールに従ってください。

---

## 1. コンパイル環境

- **コンパイラ**: Overleaf「LaTeX」（platex + dvipdfmx）
- **クラス**: `jarticle`（`myjarticle2`等の独自クラスは使わない）
- **必須パッケージ**: `graphicx[dvipdfmx]`, `amsmath`, `geometry[dvipdfmx]`
- **用紙**: A4 / 2段組 / 余白上下左右20mm / 段間2zw
- **本文**: 9pt / 行送り14.4pt（約50行/頁）

---

## 2. 章・節の番号スタイル

| レベル | 表示例 | LaTeX |
|---|---|---|
| 章 | `4．` | `\section` |
| 節 | `4.1` | `\subsection` |
| 項 | `4.1.1` | `\subsubsection` |

- 番号と見出しの間に `0.5zw` の空白を入れる
- `（１）（２）` 形式は使わない

---

## 3. 図・表・式の表記ルール（最重要）

### 3.1 名称の表記

| 要素 | 本文中の参照 | キャプション接頭辞 |
|---|---|---|
| 図 | `Fig.\ref{fig:xxx}` | `Fig.` （`\figurename`で設定済み） |
| 表 | `Table\,\ref{tab:xxx}` | `Table~` （`\tablename`で設定済み） |
| 式 | `Eq.(\ref{eq:xxx})` | — |

- 「図1」「表1」「式(1)」のような日本語表記は使わない
- `Fig.` の後のスペースは `\ref` が自動で番号を出すのでそのまま
- `Table` の後は `\,`（細いスペース）を入れる

### 3.2 掲載順序（厳守）

```
本文：「〜を Fig.1 に示す．」  ← まず明示（予告）
↓
\begin{figure}[!ht]           ← 直後にフロート環境
  ...
\end{figure}
↓
本文続き：結果の解説          ← フロートの後に考察
```

- **必ず「〜を Fig.X に示す」と予告してからフロートを配置する**
- フロートより前に結果の解説を書かない
- 数式も同様：「〜を Eq.(1) で定める．」→ `\begin{equation}` → 解説

### 3.3 フロート配置オプション

- **標準指定: `[!ht]`**（できればその場、無理なら段上部）
- `[H]` は使わない（`float`パッケージ不要）
- `figure*`（2段抜き）は使わない
- 図が参照箇所から離れすぎる場合 → figure環境ブロックごとソース内で前後に移動する（`\ref`参照なので番号は崩れない）
- `\\` による位置調整は禁止（行送りを乱すだけで図は動かない）

### 3.4 フロート緩和パラメータ（プリアンブルに記述済み）

```latex
\setcounter{topnumber}{4}
\setcounter{bottomnumber}{4}
\setcounter{totalnumber}{8}
\renewcommand{\topfraction}{0.95}
\renewcommand{\bottomfraction}{0.95}
\renewcommand{\textfraction}{0.05}
\renewcommand{\floatpagefraction}{0.9}
```

これにより1段に複数の図表を配置でき、`[!ht]`が効きやすくなる。

---

## 4. 数式

- 式番号を参照する式 → `equation` 環境 + `\label{eq:xxx}`
- 参照しない式 → `equation` 環境のままでよい（`equation*`も可）
- 数式の上下余白: `\abovedisplayskip` = 14pt（プリアンブルで設定済み）
- 本文中で式を予告してから掲載する（3.2節参照）

---

## 5. 用語の統一

| ✕ 使わない | ○ 使う | 理由 |
|---|---|---|
| GPS | GNSS | 受信機はマルチGNSS対応 |
| R_GPS | R_GNSS | 上記に合わせる |
| GPS誤差 | GNSS誤差 | 同上 |
| ground truth | 参照軌跡 | GLIMは真値ではない |
| 遮蔽率 (定義なし) | GNSS上空遮蔽率 | タイトルと対応 |

## 6. 章構成（7章・森氏スタイル）

```
1. はじめに       … 背景＋先行研究＋提案の一言（詳細は書かない）
2. 本研究の目的    … 目的文＋着目する指標の紹介
3. 仮定と問題     … 3.1 仮定（A1〜A3）／3.2 問題（P1〜P3）
4. 提案手法       … 4.1 全体構成 / 4.2 遮蔽率定義 / 4.3 動的R行列
5. 実験           … 5.1 機材 / 5.2 定点観測 / 5.3 走行実験
6. 結果と考察     … 6.1 定点観測 / 6.2 走行実験 / 6.3 仰角帯域
                     （6.1, 6.2はsubsubsectionで細分化）
7. むすび
```

- 「はじめに」で提案手法の詳細（R行列の式など）まで書かない
- 仮定は実験章ではなく3章に独立させる
- 結果章が密集するときはsubsubsectionで分割して余白を作る

---

## 7. 文体・書き方

- 敬体（です・ます）は使わない（だ・である調）
- 1文は短く。1段落3〜5文を目安
- 同じ主張を別の場所で繰り返さない（重複は統合する）
- 「選択バイアス」等の専門用語は初出時に太字 + 説明
- 脚注は `\footnotemark` + `\footnotetext` で表の直後に配置

---

## 8. 参考文献

- 引用形式: `[1]` 角括弧（`\cite`）
- 見出し: センタリング・ゴチック「参考文献」（プリアンブルで再定義済み）
- 著者名は省略せず全員記載

---

## 9. タイトル部の体裁（法政大学大学院修士論文要旨フォーマット）

- 1行目: 左「法政大学大学院修士論文要旨　理工学研究科編　(2026年7月)」右「法政大学」
- 和文タイトル: 18pt以上、センタリング
- 英文タイトル: 全大文字
- 著者: 単独著者＋「指導教員　小林 一行」行
- 共同研究者行: 不要になったら1行削除するだけの構造にする
- 英文概要: 左右5字（5zw）あけ、9pt
- Key Words: bold italic、3〜4語

---

## 10. 禁止事項チェックリスト

- [ ] `\\` による図の位置調整を使っていないか
- [ ] `figure*`（2段抜き）を使っていないか
- [ ] `[H]` や `float` パッケージを使っていないか
- [ ] 「図1」「表1」「式(1)」のような日本語表記が残っていないか
- [ ] フロートを予告（参照）する前に掲載していないか
- [ ] GPSとGNSSが混在していないか
- [ ] はじめにでR行列の式や二面性の詳細を書いていないか
- [ ] 仮定が実験章に残っていないか（3章にあるべき）
- [ ] 同じ主張が複数箇所で重複していないか
