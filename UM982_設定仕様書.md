# UM982 設定仕様書

orange2026A 搭載 GNSS 受信機（UM982 / PPP対応機）の設定リファレンス。
現状の設定スナップショットと、変更方法をまとめたもの。

作成日: 2026-07-13

---

## 1. 基本情報

| 項目 | 値 |
|---|---|
| 受信機 | Unicore UM982（デュアルアンテナ / PPP対応） |
| 接続 | USB シリアル（`/dev/ttyUSB0`） |
| ボーレート | 115200 bps |
| symlink | `/dev/sensors/GNSS_UM982`（udevルール要登録） |
| 対応測位 | 単独測位 / RTK / PPP / Moving Base heading |

## 5. 設定変更方法（Linuxから直接）

UPrecise（Windows）を使わず、ロボットに繋いだまま設定できる。

### 手順

```bash
# 1. ボーレート設定（最初に1回）
sudo stty -F /dev/ttyUSB0 115200 raw -echo

# 2. コマンド送信（末尾の \r 必須）
echo -e "GPHDT 1\r"    | sudo tee /dev/ttyUSB0 > /dev/null
sleep 0.5
echo -e "SAVECONFIG\r" | sudo tee /dev/ttyUSB0 > /dev/null

# 3. 確認
sudo timeout 3 cat /dev/ttyUSB0 | grep -E "response|GPHDT"
```

`response: OK` が返れば成功。

## 6. コマンド一覧（Unicore系書式）

| コマンド | 意味 |
|---|---|
| `GPHDT 1` | `$GPHDT`（heading）を出力 |
| `GNGGA 1` | `$GNGGA`（位置）を出力 |
| `PPPNAVA 1` | `#PPPNAVA`（PPP解）を出力 |
| `CONFIG` | モード系設定を一覧表示 |
| `LOGLIST` | ログ出力設定を一覧表示 |
| `UNLOG` | 現ポートの全ログ停止（RAMのみ） |
| `UNLOGALL` | 全ポートの全ログ停止（RAMのみ） |
| `HELP <cmd>` | コマンドの正しい書式を表示 |
| `SAVECONFIG` | 設定を不揮発メモリに保存 |
| `FRESET` | 工場出荷状態に戻す（要注意） |

**レート指定**: `1`=1Hz / `0.1`=10Hz / `0.05`=20Hz
`GPHDT ONTIME 1` のように `ONTIME` を明示する書式も可。

---

## 7. 重要な注意点

### コマンド書式は Unicore系のみ

UM982 は書式が2系統あり、この機体は **Unicore系のみ受理**する。

| 書式 | 例 | 可否 |
|---|---|---|
| NovAtel系 | `GPHDT COM2 0.1` | ✗ （OKが返るが無視される） |
| Unicore系 | `GPHDT 1` | ✓ |

→ 迷ったら `HELP <cmd>` で実機に確認。

### SAVECONFIG は必須

`SAVECONFIG` しないと電源OFFで設定が消える。
**工場出荷状態・新品に交換した場合は再設定が必要**。

### ボーレート不一致に注意

`cat` で何も出ないときは、`stty` のデフォルト9600bpsのまま。
必ず `sudo stty -F /dev/ttyUSB0 115200 raw -echo` を先に実行する。

## 8. 動作確認

```bash
# 出力メッセージの集計
sudo stty -F /dev/ttyUSB0 115200 raw -echo
sudo timeout 10 cat /dev/ttyUSB0 > /tmp/gnss_check.log
grep -oE '^\$[A-Z]+|^#[A-Z]+' /tmp/gnss_check.log | sort | uniq -c
```

`$GNGGA` と `$GPHDT` の両方が出ていれば正常。

- 屋内: `$GPHDT,,T*1B`（値が空）でも正常（fixしていないだけ）
- 屋外: heading実値が入り、`$GNGGA` の fixtype が 1 以上になる

**fixtype**（`$GNGGA` 6番目）: 0=測位なし / 1=単独測位 / 4=RTK Fix / 5=RTK Float
